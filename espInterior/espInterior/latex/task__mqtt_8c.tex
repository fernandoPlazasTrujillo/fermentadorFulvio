\doxysubsection{Descripción detallada}
Implementación de la tarea de comunicación MQTT. 

Esta tarea recibe mediciones provenientes del sistema, genera mensajes JSON y los publica en un broker MQTT.

Funcionalidades principales:\+
\begin{DoxyItemize}
\item Publicación de datos mediante MQTT.
\item Generación de payloads JSON.
\item Reintento automático de mensajes pendientes.
\item Almacenamiento local cuando no existe conectividad.
\item Recuperación de mensajes almacenados en SD.
\end{DoxyItemize}

Arquitectura:\+

queue\+\_\+mqtt \texorpdfstring{$\vert$}{|} v task\+\_\+mqtt \texorpdfstring{$\vert$}{|} +-\/-\/-\/-\/-\/-\/\texorpdfstring{$>$}{>} MQTT Broker \texorpdfstring{$\vert$}{|} +-\/-\/-\/-\/-\/-\/\texorpdfstring{$>$}{>} Archivo de pendientes (SD)

Estrategia de tolerancia a fallos:\+

Si el broker MQTT no está disponible:\+
\begin{DoxyItemize}
\item El dato no se pierde.
\item Se almacena en la tarjeta SD.
\item Se reenvía automáticamente cuando la conexión se recupera.
\end{DoxyItemize}

Mecanismos Free\+RTOS utilizados:\+
\begin{DoxyItemize}
\item Colas.
\item Notificaciones entre tareas.
\end{DoxyItemize}

Tarea principal de comunicación MQTT. 

Tarea principal de publicación MQTT.

Espera nuevas mediciones desde queue\+\_\+mqtt, genera el payload JSON correspondiente y realiza la publicación en el broker MQTT.

En caso de fallo:\+
\begin{DoxyItemize}
\item El dato se almacena en SD.
\item Se reintentará posteriormente.
\end{DoxyItemize}

Una vez finalizado el proceso, se notifica a task\+\_\+energia para continuar el ciclo operativo.
